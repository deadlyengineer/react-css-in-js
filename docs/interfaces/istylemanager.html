A style manager adds style elements when styles are registered, and removes them when they are unregistered.

Using a custom style manager allows control over how style elements are injected into the DOM.

MUST REMAIN STABLE BETWEEN VERSIONS!

register

  • register(cacheKey: string, cssText: string, replacedCacheKey?: string): void

  • Called when a new style element should be added to the DOM.

    The replacedCacheKey is the key of the style that the new style is derived from, which should be used to insert the new style in proximity to the previous style, so that precedence is not drastically affected when a style is mutated.

    Parameters

    • cacheKey: string
    • cssText: string
    • Optional replacedCacheKey: string

    Returns void
